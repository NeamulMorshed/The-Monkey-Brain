# Problem statement

> One sentence. User + need + insight. **No solution in it.**

**<Persona>** needs a way to **<need / job>** because **<insight from research>**.

- **Evidence:** cite the discovery findings ([[research-page]]) and journey pain points.
- **Success looks like:** the observable change if we solve this (feeds the spec's ACs).
- **Non-goals:** what this explicitly does *not* try to solve.

# How-Might-We questions

Reframe each key pain as an open, optimistic, appropriately-scoped question — not so broad it's
vague, not so narrow it smuggles in one solution.

- HMW **<verb>** … so that <persona> can <outcome>?
- HMW … ?
- HMW … ?

**Prioritize:** star the 1–3 HMWs worth ideating on now. These become the prompts for the
ideation phase (Crazy 8s / SCAMPER / dot-voting in `data/methods.md`).
